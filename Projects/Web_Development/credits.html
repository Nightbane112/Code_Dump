Security Considerations

This website is not without its flaws. One of the major flaws is it does not impose any access restriction to the administration page. This vulnerability not only allows malicious users to access private and confidential information of clients which may result tampering with data or lost of data altogether in addition to the violation of the Data Privacy Act. Furthermore, users could repetitively make connections to the database hampering the experience of other users and often resulting a Denial-Of-Service if prolonged. A countermeasure to alleviate this is by simply adding a login mechanism to only allow users with administrative privilege access or make changes to the website. Moreover, the login system can be better improved with logging any actions made by the admin especially transactions involving making modification to the website. When used together with a firewall, security of the website can be strengthen even further as it can be used to prevent offenders from accessing over the network.

Besides the need for privileged access, a proper and throughout error-checking process is urgently needed for form validation. Currently, any queries to the database are pass through to the server using PHP as an intermediary and SQL as for the database. As SQL is very prone to errors, any values received from a form will need some error-checking and escaping characters before sent into an SQL query string. While this method can avoid some cases of SQL injections if done correct, a better choice is to use parametrized or prepared statements. In contrast to the escaping characters function, prepared statements can be reused multiple time with ease and escaping characters are done by the PHP database driver. The object-relational mapping libraries in database driver helps produce the object-oriented code hence avoiding the use of SQL code.
